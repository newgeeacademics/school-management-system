# Deployment Guide — Render + Neon

This project uses **Neon** (PostgreSQL) for the database and **Render** for hosting the Spring Boot backend.

## Architecture

```
Frontend (Vite)  →  Render Web Service (classroom-backend)  →  Neon PostgreSQL
     :5173              classroom-backend.onrender.com              neondb
```

### Get your connection string

1. Open [Neon Console](https://console.neon.tech)
2. Select project **school-management-system**
3. Go to **Dashboard → Connect**
4. Copy the **pooled connection string** (recommended for Render)
5. It looks like: `postgresql://user:pass@ep-....neon.tech/neondb?sslmode=require`

> **Never commit this URL to git.** Set it only in Render environment variables.

## 2. Deploy backend on Render

### Option A — Blueprint (recommended)

1. Push this repo to GitHub
2. Go to [Render Dashboard](https://dashboard.render.com) → **New → Blueprint**
3. Connect repo: `newgeeacademics/school-management-system`
4. Render reads `render.yaml` and creates the `classroom-backend` service
5. In the service **Environment** tab, set:

| Key | Value |
|-----|-------|
| `DATABASE_URL` | Your Neon pooled connection string |
| `APP_CORS_ALLOWED_ORIGINS` | Your frontend URLs, comma-separated (e.g. `https://my-app.onrender.com,http://localhost:5173`) |
| `APP_JWT_SECRET` | Auto-generated by Render, or set a long random string |

6. Deploy. Your API will be at: `https://classroom-backend.onrender.com`

### Option B — Manual

1. **New → Web Service** → connect GitHub repo
2. **Root Directory:** `backend`
3. **Runtime:** Docker
4. **Environment variables:** same as above
5. **Health Check Path:** `/health`

## 3. Verify deployment

```bash
# Health check (no auth)
curl https://classroom-backend.onrender.com/health

# Login
curl -X POST https://classroom-backend.onrender.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@classroom.com","password":"admin123"}'
```

## 4. Connect frontends

Set in each app's `.env` (or Render env vars for static sites):

```env
VITE_API_URL=https://classroom-backend.onrender.com
VITE_BACKEND_BASE_URL=https://classroom-backend.onrender.com
```

| App | Env file |
|-----|----------|
| Root / classroom-app | `.env` |
| admin-app | `admin-app/.env` |
| user-portal-app | `user-portal-app/.env` |

---

## 5. Local development

```bash
cd backend
mvnw.cmd spring-boot:run
# API: http://localhost:8080 (H2 in-memory DB)
```

To use Neon locally:

```bash
set SPRING_PROFILES_ACTIVE=production
set DATABASE_URL=postgresql://...your-neon-url...
set APP_JWT_SECRET=local-dev-secret-key-min-256-bits-long
mvnw.cmd spring-boot:run
```

---

## Environment variables reference

| Variable | Required (prod) | Description |
|----------|-----------------|-------------|
| `SPRING_PROFILES_ACTIVE` | Yes | Set to `production` on Render |
| `DATABASE_URL` | Yes | Neon PostgreSQL connection string |
| `APP_JWT_SECRET` | Yes | JWT signing key (min 256 bits) |
| `APP_CORS_ALLOWED_ORIGINS` | Yes | Comma-separated allowed frontend origins |
| `PORT` | Auto | Set by Render (default 8080) |
